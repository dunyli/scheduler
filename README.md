## Системный монитор и симулятор планировщика процессов
### Описание проекта
Данный проект представляет собой учебно-практическую утилиту для просмотра загруженности системы и симуляции алгоритмов планирования процессов. 

Программа сочетает в себе два режима работы:

1) Режим мониторинга системы (аналог top/htop) - отображает реальные процессы Windows,
загрузку CPU и использование памяти в реальном времени с интерактивным управлением.
2) Режим симулятора планировщика - демонстрирует работу алгоритмов планирования процессов (FIFO, Round Robin, Priority)
на виртуальных процессах с визуализацией и статистикой.

#### Режим 1: Мониторинг системы
Режим показывает реальное состояние системы в терминале с обновлением по нажатию клавиш.

Отображаемая информация:

Использование памяти (% и MB/GB)

Список процессов (до 50, отсортированный по памяти):

- PID
- Имя процесса
- Использование памяти (KB)
- Приоритет (Idle, Below, Normal, Above, High, Real)
- Права администратора (Да/Нет)

#### Режим 2: Симулятор планировщика
Режим позволяет выбрать алгоритм планирования и увидеть его работу на виртуальных процессах.

Отображаемая информация:

- Информация о процессах: PID, имя, время прибытия, время выполнения, приоритет, статус
- Расписание выполнения: таблица с временными интервалами
- Сводка по времени: общее время, время выполнения, время простоя
- Статистика процессов: время ожидания, время оборота
- Общая статистика: средние значения, экстремумы

### Теоретическая часть
В проекте реализованы и исследованы следующие алгоритмы планирования:

#### 1. FIFO (First In First Out)

Тип: Невытесняющий

Принцип: Процессы выполняются в порядке поступления

Применение: Пакетная обработка, очереди печати

Особенности: Простой, предсказуемый, но может вызывать "эффект конвоя"

#### 2. Round Robin (Круговая очередь)

Тип: Вытесняющий

Принцип: Каждому процессу выделяется квант времени

Применение: Интерактивные системы, системы разделения времени

Особенности: Справедливое распределение CPU, частые переключения контекста

#### 3. Priority Scheduling (Приоритетное планирование)

Тип: Вытесняющий (с приоритетным ускорением)

Принцип: Процессы с более высоким приоритетом выполняются первыми

Применение: Системы реального времени, мультимедиа

### Описание модулей

1) process - Создание, копирование, управление процессами
2) scheduler_core - Базовые структуры и функции планировщика
3) fifo - Реализация алгоритма FIFO
4) round_robin - Реализация алгоритма Round Robin
5) priority - Реализация приоритетного планирования
6) visualizer - Визуализация в терминале (таблицы, статистика)
7) monitor - Реальный мониторинг системы (Windows API)
8) test	- Автоматические тесты алгоритмов


### Описание тестов программы
В программе реализованы 4 автоматических теста для проверки корректности работы алгоритмов планирования. Тесты находятся в файле test.cpp и запускаются при выборе режима 3 в главном меню.

Все тесты используют предопределенные наборы процессов и выводят результаты в консоль с цветовым выделением.

#### Тест 1: Базовый тест (test_basic)
Проверяет работу всех трех алгоритмов на простом наборе из 3 процессов.

Что выводит:

Для каждого алгоритма (FIFO, Round Robin, Priority):
- Информацию о каждом процессе: ожидание, оборот
- Среднее время ожидания

#### Тест 2: Проверка приоритетного вытеснения (test_priority_preemption)
Проверяет, что процесс с более высоким приоритетом вытесняет процесс с низким приоритетом.

Что выводит:
- Время завершения и приоритет каждого процесса
- Результат проверки: работает корректно или есть ошибка


#### Тест 3: Проверка квантования в Round Robin (test_round_robin_quantum)
Проверяет, что процессы правильно вытесняются по истечении кванта времени.

Что выводит:

- Время завершения каждого процесса
- Время ожидания каждого процесса
- Время оборота каждого процесса
- Результат: все процессы завершены или нет

#### Тест 4: Стресс-тест (test_stress)
Проверяет работоспособность с большим количеством случайных процессов.

Тестовые данные:
- 30 случайных процессов
- Случайные параметры: прибытие (0-9), выполнение (2-7), приоритет (0-3)

Что выводит:
Количество сгенерированных процессов

Для каждого алгоритма (FIFO, Round Robin, Priority):
- Среднее время ожидания
- Среднее время оборота
- Результат прохождения теста


### Установка и сборка

#### Требования

- Операционная система: Windows
- Компилятор: Visual Studio 2019/2022 или MinGW
- Библиотеки: Стандартные библиотеки C/C++

#### Сборка через командную строку
// Для MinGW
g++ -std=c++11 -o scheduler.exe scheduler.cpp process.cpp scheduler_core.cpp fifo.cpp round_robin.cpp priority.cpp visualizer.cpp monitor.cpp test.cpp -lpsapi -lwinmm

// Для MSVC
cl /EHsc /Fe:scheduler.exe scheduler.cpp process.cpp scheduler_core.cpp fifo.cpp round_robin.cpp priority.cpp visualizer.cpp monitor.cpp test.cpp

#### Запуск программы
./scheduler.exe
